# 服务器部署与多机迁移极简手册

这份手册面向当前仓库的实际用法：

- 单机 Docker 部署
- 多台服务器共享同一个图床
- 使用 `/sync` 页面同步上传记录，避免重复上传

## 适用场景

- 你已经有 1 台正在运行的服务器 A
- 现在要新增 1 台服务器 B
- 两台服务器都部署这个项目
- 两台服务器都会上传图片到同一个图床

目标是：

1. 快速部署新服务器
2. 把 A 的上传记录迁移到 B
3. 让 B 后续上传时直接复用已有图床 URL

## 一、先部署第一台服务器

在服务器上执行：

```bash
mkdir -p data downloads
docker compose up -d --build
```

第一次启动后：

- 应用默认地址：`http://服务器IP:8000`
- 配置文件位置：`data/config.yaml`
- 数据库位置：`data/wallpaper.db`

如果要改图床、代理、同步策略，直接改：

```bash
sed -n '1,160p' data/config.yaml
```

改完重启：

```bash
docker compose restart
```

## 二、配置源服务器 A 的同步导出

打开服务器 A 的 `/sync` 页面，建议至少配置：

- 本机同步密钥
- 来源白名单
- 每分钟请求上限

推荐策略：

- `auth_token`：填一个强随机字符串
- `allowed_sources`：只放目标服务器 B 的出口 IP
- `export_rate_limit_per_minute`：先用 `60`

如果你更习惯手改配置，也可以在 `data/config.yaml` 里写：

```yaml
sync:
  auth_token: "replace-with-a-strong-token"
  allowed_sources:
    - "服务器B出口IP"
  export_rate_limit_per_minute: 60
```

## 三、部署第二台服务器 B

在服务器 B 上执行：

```bash
mkdir -p data downloads
docker compose up -d --build
```

启动后访问：

```text
http://服务器B:8000/sync
```

## 四、从 A 一键迁移到 B

在服务器 B 的 `/sync` 页面：

1. 填写服务器 A 地址
2. 填写服务器 A 的同步密钥
3. 先点“测试远程连接”
4. 测试通过后点“一键从远程服务器迁移”

测试时建议确认：

- 远程服务可达
- 对端识别到的来源 IP 正确
- 对端注册表总量 / 可导出数量正常
- 对端白名单与限流配置符合预期

迁移完成后：

1. 看“注册表记录”数量是否增加
2. 看“最近操作”里是否出现迁移记录
3. 随机选一张历史已上传图片验证是否直接复用 URL

## 五、如果两台服务器不能互通

改用手动迁移：

### 服务器 A

1. 打开 `/sync`
2. 选择是否按 Profile / 格式筛选导出
3. 点击导出 JSON

### 服务器 B

1. 打开 `/sync`
2. 选择导出的 JSON
3. 先看同步包预览
4. 再执行导入

## 六、迁移后建议做一次重复项清理

如果这个项目已经多次导入、反复回填过历史数据，建议在 `/sync` 页面执行：

1. 扫描注册表重复项
2. 观察是否有 URL 冲突组
3. 确认无误后执行一键合并重复项

这个步骤不是必须，但能让注册表更干净，后续排障更容易。

## 七、最常见的报错

### `403 同步密钥无效`

说明：

- 目标服务器传过去的密钥不对

检查：

1. 源服务器 `/sync` 页面的同步密钥
2. 目标服务器填写的“远程迁移密钥”

### `403 来源 xxx 不在同步白名单内`

说明：

- 源服务器拒绝了当前请求来源

检查：

1. 源服务器 `allowed_sources` 是否包含目标服务器出口 IP
2. Nginx / 反向代理是否正确转发 `X-Forwarded-For` 或 `X-Real-IP`
3. `/sync` 页面里“对端识别到的来源”是否符合预期

### `429 同步接口请求过于频繁`

说明：

- 命中了导出限流

处理：

1. 等 1 分钟后再试
2. 或适当提高 `export_rate_limit_per_minute`

### 测试连接正常，但迁移失败

说明：

- `handshake` 能通，但 `export` 可能被单独拦住

优先检查：

1. 同步密钥
2. 白名单
3. 限流
4. 反向代理配置

### 导入成功，但后续还是重复上传

优先检查：

1. `/sync` 页面里“注册表记录”是否真的增加
2. 这批图片是否具备相同的 `resource_id` / `sha256` / `md5`
3. 是否导入到了错误的服务器实例

## 八、建议的生产实践

- 优先走内网迁移，不要长期暴露公网同步接口
- 公网暴露时，至少同时启用：
  - 同步密钥
  - 来源白名单
  - 导出限流
- 每次大批量迁移前先点“测试远程连接”
- 每次导入后看“最近操作”确认已被后端记录
- 不要直接同步整个 `wallpaper.db`，优先同步上传记录注册表

## 九、最短命令清单

### 启动

```bash
mkdir -p data downloads
docker compose up -d --build
```

### 看日志

```bash
docker compose logs -f
```

### 改配置后重启

```bash
docker compose restart
```

### 停止

```bash
docker compose down
```
